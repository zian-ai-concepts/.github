# Contributing

Beiträge zu ZIAN-Projekten sollen klar, nachvollziehbar und wartbar sein.

## Grundsätze

- Beschreibe Änderungen präzise.
- Halte Dokumentation und Code konsistent.
- Verwende das ZIAN Brand System für README-Grafiken, Präsentationen und Projektkommunikation.
- Vermeide unnötige visuelle Abweichungen in öffentlichen Repositories.

## Pull Requests

Ein Pull Request sollte enthalten:

- kurze Zusammenfassung
- relevante Screenshots oder Beispiele, falls UI betroffen ist
- Hinweise auf offene Punkte oder Risiken
